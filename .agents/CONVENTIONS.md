# Conventions - Vrata

## Mandatory Patterns

- **DRY, KISS, elegant** — every abstraction must earn its place. If it can be done with
  less code without sacrificing clarity, do it with less code.
- **Fault isolation between entities** — a validation error in one route, middleware,
  destination, or any other entity must NEVER prevent the rest from working. The broken
  entity is skipped with a visible `slog.Error` log, and the xDS snapshot is built with
  everything else. This applies to xDS translation of every entity type.
- **Explicit dependency injection** — shared dependencies (store, xDS server, logger, config)
  are passed via a `Dependencies` struct. No package-level globals, no `init()` side effects.
- **Error bubbling** — functions return errors to their caller. Only handler-level code
  (and `main`) handles errors by logging and/or converting them to responses. Never discard
  an error with `_` unless there is an explicit, commented reason.
- **Human-readable API errors** — HTTP error responses must be JSON with a clear `message`
  field. Raw Go error strings must never reach the client.
- **All symbols documented** — every exported type, function, method, and constant gets a
  Go doc comment. Unexported symbols that are non-obvious also get one. swag-go v2
  annotations are required on all HTTP handler functions.
- **Structured logging via slog** — use `slog.Info`, `slog.Error`, etc. with key-value
  attributes. No `fmt.Println`, no `log.Printf`. Log level and format (text/JSON) are
  controlled by config.
- **Config via YAML + env expansion** — `os.ExpandEnv` is applied to raw YAML bytes before
  unmarshalling. All configurable string values support `${ENV_VAR}` syntax.
- **Atomic duplicate detection** — route uniqueness checks (match rule collision within a
  group) must be performed atomically with the write inside the store. Never check then write
  as separate steps.
- **English everywhere** — code, comments, commit messages, variable names, doc strings,
  Makefile targets. No exceptions.
- **Model is proxy-agnostic** — domain types in `internal/model/` must never reference
  Envoy protobuf types, xDS field names, or proxy-specific concepts. The translation
  from model to xDS lives exclusively in `internal/xds/`.

## Forbidden Patterns

- **No external router libraries** (Gin, Echo, Chi, etc.) unless there is a concrete,
  documented, justified reason. `net/http` is the baseline.
- **No global state** — no package-level variables that hold mutable runtime state.
- **No silent error swallowing** — `if err != nil { return }` without propagating the error
  is forbidden. Always wrap with context: `fmt.Errorf("doing X: %w", err)`.
- **No raw Go errors in HTTP responses** — never write `err.Error()` directly into a JSON
  response body. Map to a user-facing message first.
- **No mixed swag-go annotation styles** — always use swag-go v2 format.
- **No manual editing of `docs/`** — the `docs/` directory is fully generated by `swag`.
- **No Envoy types in model/** — the model package must remain proxy-agnostic. All
  protobuf imports and xDS-specific logic belongs in `internal/xds/`.
- **No partial xDS updates** — always rebuild the full snapshot. Do not implement
  incremental xDS (delta ADS) without an explicit decision.

## Naming

- **Files**: `snake_case.go`. One primary type or concern per file.
- **Packages**: short, lowercase, no underscores (`config`, `model`, `store`, `api`, `xds`, `gateway`).
- **Types and interfaces**: `PascalCase`. Interfaces named by behavior: `Store`, not `IStore`.
- **Functions and methods**: `camelCase` (unexported), `PascalCase` (exported).
- **Constants and config keys**: `PascalCase` for Go consts; `camelCase` or `snake_case` for YAML keys.
- **HTTP handler functions**: named `HandleVerbResource`, e.g. `HandleCreateGroup`, `HandleListRoutes`.
- **Error variables**: `ErrSomething`, e.g. `ErrDuplicateRoute`, `ErrGroupNotFound`.
- **xDS naming**: `clusterName(destinationID)` → `"dest_" + id`, `routeConfigName(listenerID)` → `"rc_" + id`.

## Error Handling

- Wrap errors with context at every layer: `fmt.Errorf("creating route: %w", err)`.
- Define sentinel errors in `internal/model` or the relevant package for known failure cases.
- The API must never return a 500 for a client mistake. Validate input early and return 400/409/422.
- Panics in handlers are caught by the recovery middleware and converted to a 500 JSON response.

## Logging

- Use `slog` from the standard library exclusively.
- Format is controlled by config: `console` (human-readable text) or `json`.
- Log level is controlled by config: `debug`, `info`, `warn`, `error`.
- Every HTTP request must be logged by the logger middleware with: method, path, status code,
  duration, and (if applicable) error message.
- Do not log sensitive data (tokens, passwords, full request bodies by default).

## Testing

Unit tests exist for API handlers, store, model, config, raft, k8s watcher, session.
xDS translator tests are pending. E2E tests with Envoy are pending.

## Configuration

- Config is loaded from the path given by `--config` flag.
- `os.ExpandEnv` is applied to raw YAML bytes before parsing.
- The `Config` struct lives in `internal/config/config.go` and is the single source of truth.
- Config is passed to components at startup via the `Dependencies` struct.
- `config.yaml` in the repo root documents every available key with inline comments.
